Changelog · what changed vs today

  • Removed the round letter avatar on every row — pure decoration; was costing 54px of horizontal space.
  • Removed the inline status label chip (CONFIRMED / UNAVAILABLE / SUBSTITUTED) — the left stripe + the price line already say everything the chip said.
  • Removed the orange pill background & pen ✎ icon on vessel notes — pill chrome was loud; pen icon read as "edit"-ish, not "note".
  • Renders notes as italic quoted lines directly under the item name. Three colour-coded variants: muted grey (vessel set), terracotta with Sub: prefix (substitute), and navy (supplier wrote back).
  • Replaces the slim round chat-bubble with a sturdier square chat icon (rectangle + tail + two lines). Filled terracotta when a note is attached, outline when empty.
  • Adds category section headers — DM Serif Display title + small tally pill. Lets the supplier scan a 17-item order by domain (galley vs caviar vs greens) instead of reading top-to-bottom.
  • Drops the category breadcrumb that used to live under each item — redundant once the section heading carries it.

Open questions for you

Q1. Collapsible category groups? Tap the category header to fold the lines inside. Useful when you have 17+ items and want to scan completed sections quickly. Default: all expanded.
Q2. Section-level mini-totals? Show €182.00 · 1 awaiting quote on the right of the category header so the chief can see per-category cost at a glance. Helps when reviewing drinks-vs-veg-vs-luxury spend.
Q3. Drop the Size column on supplier orders? It's always — on every row in the current screenshot. If it's hardly ever populated for supplier orders, recover the 60px for a wider Item column.
Q4. Where does the "Edit" button on completed lines belong? Today it sits inline on every confirmed/sub/unavailable row. Options: keep inline (current), move into a row-level overflow menu (⋯), or only show on hover. My pick: hover-only — cuts visual noise dramatically on long orders.
