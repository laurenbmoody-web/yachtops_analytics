My recommendation

Go square (C) for this department page. With a dozen departments — many at 0 items — the current wide cards (A) burn a lot of horizontal space and make empty departments feel heavier than they are. Square tiles read as a calm, scannable grid of "places," which is exactly what this level is: a launchpad, not a data table.

Keep the meta and quantity detail for the deeper pages, where tiles carry item counts, low-stock and expiry signals — there the compact card (B) earns its width. So: square at the top level, compact cards once you're inside a department.

Option D (cover) looks great but its value is images; until departments/folders carry a photo, the icon panel is doing a card's job with more chrome. Worth revisiting if we add folder cover images later.
